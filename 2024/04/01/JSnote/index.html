JS，专为与网页交互而设计的脚本语言；诞生于1995年

  1. ECMAScript：核心语言，由ECMA-262定义

    • 组成部分：语法 / 类型 / 语句 / 关键字 / 保留字 / 操作符 / 对象

  3. BOM(Browser Object Model)：浏览器对象模型，提供与浏览器交互的方法和接口

变量 var xx

  • 是松散类型，可以用来保存任何类型的数据。
  • 未初始化时，有一个特殊的值 undefined
  • 全局变量很难维护，很难被收回

    基本类型，简单的数据段

  • 原始值 undefined / null / boolean / number / string / symbol
  • 在内存中占据固定大小空间，被保存在栈内存中

    引用类型，可能由多个值构成的对象

  • 值为保存在内存中的对象，保存在堆内存中，不允许直接访问，按引用访问
  • 可动态地添加属性
  • 当做变量复制时，这个值的副本实际上是一个指针，指向存储在堆中的同一个对象

    执行上下文

  • 变量或函数的上下文决定了它们可以访问哪些数据以及行为
  • 每个上下文都有一个关联的变量对象
  • 上下文中的代码在执行时，会创建变量对象的一个作用域亮
  • 上下文之间连接是线性的、有序的

    变量声明

  • var声明会被拿到函数或全局作用域顶部，位于作用域中所有代码之前，这个叫做变量提升
    • 会让同一作用域的代码不用考虑变量是否已声明就可以直接使用
  • let 块级作用域，由最近的一对包含{}来界定
    • 同一作用域不能声明两次
    • 适合在循环中声明迭代变量，不像var声明会泄漏到循环外部
      • 将其重新绑定到循环的每个迭代，确保使用上一个循环结束时的值重新进行赋值
    • 严格来说，let声明也会被提升，但由于暂时性死区，实际上不能在声明前使用let变量
  • const关键字，声明同时必须初始化，且不能再被重新赋新值
    • 只应用到顶级原语或对象，对象的键不受限制
    • 运行时编译器可以将所有实例换成实际的值，不会通过查询表进行变量查找

    闭包，是一个有权访问另一个函数作用域中变量的函数

  • 常见方式：在一个函数内部创建另一个函数

  • 闭包作用域链包含自己的、包含函数的和全局的

  • 在作用域链中查找变量，作用域链本质上是一个指向变量对象的指针列表
    | 本函数活动对象 | 0 |
    | 外部函数的活动对象 | 1 |
    | 外部函数的外部函数的活动对象 | 2 |
    …
    | 全局执行环境 | 终点 |

    • 一般来讲，函数执行完毕后，局部活动对象就会被销毁，内存中仅保存全局作用域
    • 而闭包则会使内部定义的函数将包含函数的活动对象添加到它的作用域链中
      • 当被引用的函数返回后，其执行环境的作用域链会被销毁，但活动对象仍会留在内存中
    • 过度使用闭包可能导致内存占用过多

  • 闭包只能取得包含函数中任何变量的最后一个值，因为它们引用的是一个活动对象里的变量

    • 闭包保存的是整个变量对象，而不是某个特殊变量

  • 匿名函数的执行环境具有全局性，this通常指向window

    • 要想访问作用域中的arguments对象，必须将该对象的引用保存到另一个闭包能访问的变量中

  • 闭包会引用包含函数的整个活动对象，有可能会造成引用无法解除，占用内存

    • 创建闭包必须要维护额外的作用域，过度使用可能会占用大量内存

  • 块级作用域，私有作用域，限制向全局作用域中添加过多变量和函数

    function(){}() 这是函数声明，不能加()会导致错误

  • 闭包还可以用于在对象中创建私有变量、私有函数

    • 有权访问私有变量的共有方法叫 特权方法
      • 单例模式、构造函数模式、原型模式
    • 在原型prototype上定义公有方法 例如get() set()
      • 但在一个实例上调用时会影响所有实例，可能会导致返回相同的值

    原型，行为重用通过复制已存在的原型对象来实现

  • 每个函数都有一个prototype属性，该属性是一个指针指向一个对象

  • Object内建对象是最高级父级对象

  • 该对象用途是包含可以由特定类型的所有实例共享的属性和方法

  • 好处是可以让所有对象实例共享它所包含的属性和方法

  • 所有原型对象会自动获得一个constructor（构造函数）属性，包含一个指向prototype属性所在函数的指针

  • isPrototype() Object.getPrototypeOf()

  • 当为对象实例添加一个属性时，就会屏蔽原型对象中保存的同名属性，即会阻止我们访问原型中的属性，但不会修改那个属性

    • 自身属性优先级高于其原型对象中同名属性

  • 使用hasOwnProperty()判断哪个是实例属性，哪个是原型 属性只在属性存在于实例中时才返回true

  • in (“xxx” in p1) 只要能访问到就为true / for in （能通过对象访问的、可枚举的属性）实例和原型中的属性都包括

  • Object.keys() 返回包含所有可枚举属性的字符串数组，Object.getOwnPropertyNames()无论是否枚举都返回

  • 使用对象字面量形式创建时，相当于重写了prototype对象，constructor属性就会变成新的，指向了Object构造函数

  • 在原型中查找值的过程是一次搜索，对原型对象做的任何修改都能够立即从实例上反映出来

  • 原型陷阱：当重写了整个原型对象时，就切断了跟先前已经创建的实例对象之间的联系，实例引用的依然是最初的原型

    问题

  • 推荐使用构造函数+原型模式组合使用

    • 即所有自定义类型都会继承toString()、valueOf()等默认方法
  • 用instanceof() isPrototypeOf() 确定原型和实例等关系
  • 实现继承时，不能使用对象字面量创建原型，这样就重写原型链，切断了相互间的关系
  • 在创建子类型实例时，不能向超类型的构造函数中传递参数，解决方法如下

    借用构造函数，在子类型构造函数内部调用超类型构造函数

  • xxx.call(this，参数值)

    组合继承

  • 使用原型链实现对原型属性和方法的继承
  • 通过借用构造函数来实现对实例属性的继承
  • 这样既在原型上定义方法实现函数复用，又能保证每个实例都有它自己的属性

    原型式继承

  • 借助已有对象创建新对象

    寄生式继承

  • 借用构造函数来继承属性，通过原型链等混成形式来继承
  • ES5 Object.create(person, object) 规范了原型是继承
    • object { xxx: { vlaue: ‘’} }

性能

JavaScript 引擎会在编译阶段进行数项的性能优化。
其中有些优化依赖于能够根据代码的词法进行静态分析，并预先确定所有变量和函数的定义位置，才能在执行过程中快速找到标识符。
但如果引擎在代码中发现了eval()或with()，它只能简单地假设关于标识符位置的判断都是无效的，因为无法在词法分析阶段明确知道 eval()会接收到什么代码，这些代码会如何对作用域进行修改，也无法知道传递给with用来创建新词法作用域的对象的内容到底是什么。
最悲观的情况是如果出现了eval()或with()，所有的优化可能都是无意义的，因此最简单的做法就是完全不做任何优化。

  • 使用任何一个机制都将导致代码运行变慢

    编译

    | 分词 / 词法分析 | var / a / = / 2 |
    | 解析 / 语法分析| AST抽象语法树 |
    | 代码生成 |
  • 大部分编译发生在代码执行前的几微秒甚至更少时间内
  • 引擎、编译器、作用域
  • LHS 赋值 / RHS 取值
  • 包含变量和函数所在内的所有声明都会在任何代码被执行前首先处理
    • 首先这个分为2部分：
      • var a 在编译阶段进行，即声明提升了
      • a = 2 会被留到原地等待执行阶段时才进行赋值操作
    • 先声明后赋值
    • 函数声明会比变量先提升

      模块

  • 两个特征：
    • 为创建内部作用域而调用了一个包装函数
    • 包装函数的返回值必须至少包括一个对内部函数的引用，这样就会创建涵盖整个包装函数内部作用域的闭包

动态作用域

调用栈

  • 为了到达当前执行位置所调用的所有函数，可以当作一个函数调用链
  • 非严格模式下默认绑定会绑定到全局对象
  • 调用位置是否有上下文 a.b.c，隐式绑定会把函数c中调用的this绑定到这个上下文对象中，对象属性引用链中只有最后一层（即b）影响调用位置
    • 被隐式绑定到函数会丢失绑定对象，就会引用默认绑定
  • 显式绑定，call() / apply() / 硬绑定 bind() (Function.prototype.bind）
    • 把 null / undefined 作为this的绑定对象传入 call/apply /bind，这些在调用时会被忽略，实际应用默认绑定规则
  • new 创建的函数，this绑定的是新创建的对象
  • 优先级： new > 显式绑定 > 隐式绑定
  • 箭头函数的绑定无法被修改

  • JS中实际上不存在所谓的 构造函数，只有对于函数的 构造调用

  • 创建一个全新对象

  • 新对象会被执行 原型 连接

  • 新对象会绑定到函数调用的this

  • 如果函数没有返回其他对象，那么new表达式的函数调用会自动返回这个新对象

  • Object.create(null)和{}很像，但是并不会创建Object.prototype这个委托

    • Object.create(null)没有原型链，所以它比{}“更空”

  • Object.create()会创建一个新对象并把它关联到我们指定的对象

    • 通过Prototype机制委托来关联对象并且避免不必要的麻烦
    • 用 new 的构造函数调用会生成 .prototype 和 .constructor 引用
