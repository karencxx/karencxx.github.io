Vue.js 使用 Flow 做静态类型检查

  • Flow 是 facebook 的 JavaScript 静态类型检查工具
    • 类型检查分成 2 种方式：
      • 类型推断：通过变量的使用上下文来推断出变量类型，然后根据这些推断来检查类型
      • 类型注释：事先注释好的类型 如x:number，Flow会基于这些注释来判断

Vue 实际上是一个类，在JS中用 Function 实现。这就是需要用 new Vue(..) 来初始化的原因

核心思想

  1. 数据驱动：视图是由数据驱动生成，不直接操作DOM，而是通过修改数据来实现。
  2. 组件化

Virtual DOM

用原生JS对象去描述一个DOM节点
借鉴了Snabbdom.js
作用：

  • 维护视图和状态的关系，可以跟踪状态变化
  • 复杂视图情况下提升渲染性能
  • 可以跨平台渲染

在Vue中，VNode是对 真实DOM 的抽象描述，有几个关键属性：标签名tag、数据data、子节点children、键值text、上下文环境context等

除了数据结构等定义外，要映射到 真实的DOM节点 上实际上还需要几个步骤：

  1. create 创建Virtual DOM树
  2. diff
  3. patch 用于渲染对比VNode变成真实DOM

生命周期

  • 钩子函数 LIFECYCLE_HOOKS
  1. init初始化
    • 合并配置
    • 初始化生命周期
    • 初始化事件中心
    • 初始化渲染
    • 初始化 data、props、computed、watcher
  2. $mount挂载vm
  3. 先编译compile
  4. render
  5. VNode

数据响应，数据劫持

两部分组成：观察者(watcher) 和 依赖收集器(Dep)

  • Vue 2.0响应 是基于ES5 的 Object.defineProperty 实现
    • 设置data后，遍历所有属性，利用defineProperty 重写 属性的 get 与 set 方法，从而完成监听数据的改变
    • 无法监听数组变化
    • 无法处理属性也为对象的情况
  • Vue 3.0响应 是基于ES6 的 Proxy 实现
  • Proxy 对象用于创建 一个对象的代理，从而实现基本操作的拦截和自定义，如属性查找、赋值、枚举、函数调用等
    • new Proxy(target, handler)
  • 优点
    • 不需要深度遍历监听
    • 对象、数组变化都能监听得到

观察者模式

  • 被观察者 subject、观察者 observe
  • 是一对多的关系，是耦合的，适合在组件内部操作

发布订阅模式

  • 订阅者 subscriber、发布者 publisher
  • 消息中心 Dep，接到指令后会根据事件类型给对应的订阅者发送消息
  • 添加消息中心，所有消息均通过消息中心，发布者与订阅者不直接相关，实现了两者解耦。更适合消息类型复杂情况
