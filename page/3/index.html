说说使用v-for时key的作用

通常我们在用v-for指令渲染列表时，如果没有定义 v-bind:key，那么会得到错误提示，
可能你会不以为然地加上就万事大吉了
但，在某些情况下还是要小心为妙

先来看一下vue官方文档

即当不使用key时，vue更倾向于 复用DOM，尝试改变 数据层面 而不是 去操作DOM层面
但文档上提出来这个模式的使用范围：

我们来看个例子

文档中提到了临时DOM状态，这个例子里我们看子组件里的input输入框
该输入框没有绑定 Vue实例 中的任何数据，也就是说他是 独立于当前应用程序的状态
如果我们在框中输入内容，那么这个内容将会被暂时存储在 临时DOM状态中

更改itemsArray数组元素顺序

我们随便输入一些到输入框中，我们通过底下的unshift first 按钮去改变顺序，在数据头部增加一组ID为11的数据

可以看到input输入框不会随着数组顺序的改变而变化
为何？这就是上面说的 就地更新，改变数组元素顺序并没有移动DOM元素，而是数据发生变化。

于是，我们把key属性加上
使用 key 时，它会基于key的变化重新排列元素顺序，并且会移除 key不存在 的元素

但结果并没有变化
由于这里的key值只是采用了当前项的索引
当数组数据改变时，index并没有变化，只是传入的item发生了改变，所以输入框的内容还是不会改变

我们改用数据项的id作为key值

id 是每项数据的唯一标识，能够确保每项的key是独特唯一的，vue能正确跟踪key变化而重新排列元素顺序

也就是说，我们应该尽可能地使用节点数据的 唯一ID
这样做Vue可以更准确地跟踪 每个节点的身份并更新组件状态，并且能够 重用 和 重新排列 现有元素

注意:

  • 不要使用 对象 或 数组 之类的 非基本类型值 作为 v-for的key
    • 接受的值为：number | string | boolean (2.4.2 新增) | symbol (2.5.12 新增)
  • 重复的 key 会造成渲染错误

跨域

跨域是浏览器基于同源策略的一种安全手段，是浏览器出于安全性做的限制

同源策略( Sameoriginpolicy )

  • 是一种约定，是浏览器最核心也最基本的安全功能
  • 用于隔离潜在恶意文件的安全机制，通常不允许不同源之间的读操作

同源

  • 协议相同 protocol
  • 主机相同 host
  • 端口相同 port

www.xxx.com 中 www是顶级域名，xxx.com是二级域名（主域名）
aaa.xxx.com 中 aaa则是分域名（子域）

url-1 url-2 允许通信 说明
http:www.example.com/a.js http:www.example.com/lab/b.js ✅ 同一域名，不同文件路径
http:www.example.com:8080/a.js http:www.example.com/c.js ❎ 同一域名，不同端口
http://www.example.com/a.js https://www.example.com/c.js ❎ 同一域名，不同协议
http://www.example.com/a.js https://192.169.2xx.2x/b.js ❎ 域名和域名对应相同的ip
http://www.example.com/a.js http://www.demo.com/c.js ❎ 主域不同
http://www.example.com/a.js http://x.example.com/c.js ❎ 子域不同

当没有这个限制时，可能你会碰到

  • Cookie劫持，被恶意网站窃取数据
  • 更易受到XSS、CSRF攻击
  • 无法隔离潜在恶意文件

也就是说，非同源的请求就会产生跨域

解决方法

  • JSONP
  • document.domain + iframe
  • window.name + iframe
  • location.hash + iframe
  • postMessage
  • CORS
  • Nginx代理
  • nodejs中间件代理
  • webSocket协议

JSONP（JSON with Padding）

最终效果

优点：兼容性好，可兼容老式浏览器
缺点：需要服务端支持且只支持GET请求

CORS跨域资源共享（Cross-origin Resource Sharing）是一个系统

在vue项目中，一般关注 CORS / proxy

如果服务端未使用*, 而是指定了一个域，那么为了向客户端表明服务器的返回会根据Origin请求头而有不同, 即对不同源返回不同的内容
必须在Vary响应头中包含Origin

Simple requests（简单请求，不触发CORS预检）

使用方法： GET / HEAD / POST
Content-type值的限制

  • text/plain
  • multipart/form-data
  • application/x-www-form-urlencoded

Preflighted requests（预检请求）

基于OPTIONS方法进行过滤，发起一个预检请求到服务器，以获知是否允许该实际请求
作用：避免跨域请求对服务器的用户数据产生未预期的影响

OPTIONS请求通过Access-Control-Request-Method告知实际请求使用的方法
服务端返回响应Access-Control-Allow-Methods表示允许客户端使用的方法发起请求

实现CORS的关键是服务器

优点：支持所有请求方法
缺点：兼容性比JSONP差

运行一个通过这个服务与另一个网络终端进行非直接的连接

  1. 如果是通过vue-cli搭建工具，可通过webpack起一个本地服务器作为请求代理对象
    但上线时web应用和接口服务器不在一起，还是会跨域

  2. 服务器实现代理请求转发，以express框架为例

  3. 通过配置nginx实现代理

PostMessage

是HTML5 XMLHttpRequest Level 2中的API
可以实现跨文档通信
可与Websocket、Service Work进行通信

ts基础类型

TypeScript 是 JavaScript 的类型的超集，它可以编译成纯 JavaScript。
即TS包含了JS的所有特性

常见类型

元祖类型 Tuple

表示一个已知元素数量和类型数组，各元素类型不必相同

当访问越界元素时，会使用联合类型替代

🚫 ts 3.1版本 之后，访问越界元素会报错

枚举类型

默认情况下，元素值从0开始，也可手动赋值

提供便利：由枚举值得到属性名

any类型 任意类型

对现有代码进行改写时，any类型十分有用

  • 允许你在编译时可选择地包含或移除类型检查
  • 并且当只知道一部分数据的类型时，any 类型也是有用的

unknown类型 类型不确定的变量

  • unknown比any在类型上更安全
  • 任意类型的值可以赋值给unknown
  • 但unknown类型的值只能赋值给 unknown 或 any

void 无任何类型

通常用于函数，表示没有返回值
声明一个 void 变量是无意义的，只能赋给 null | undefined

null & undefined 任意类型

默认情况下 null 和 undefined 是所有类型的子类型
子类型可以赋值给父类型
像下面这个例子，null 和 undefined 赋值给 number类型 的变量

常规编译不会出错
严格编译 tsc xxx.ts –strictNullChecks 将会检查 null，从而规避一些问题

联合类型 任意类型

never 那些永不存在的值的类型

  • 是任意类型的子类型，可以赋值给任何类型
  • 别的类型都不是 never的子类型，不可赋值给 never
  • 常用于抛出一些异常 或 没有返回值的函数表达式 或 箭头函数表达式的返回值类型

返回never的函数 必须存在 无法达到的终点

object

除 number / string / boolean / symbol / null / undefined 之外的类型，非原始类型
类似object.create() API

类型断言

有时候你会遇到这样的情况，你会比 TypeScript 更了解某个值的详细信息。
通常这会发生在你清楚地知道一个实体具有比它现有类型更确切的类型。

通过类型断言这种方式可以告诉编译器，“相信我，我知道自己在干什么”。
类型断言好比其它语言里的 类型转换，但是不进行特殊的数据检查和解构。
它没有运行时的影响，只是在 编译阶段起作用

范型

  • 可以理解为类型参数化，将某种具体的类型进行参数化
  • 就像定义函数参数一样，给泛型定义若干个类型参数，并在调用时给泛型传入明确的类型参数
  • 目的在于有效约束类型成员之间的关系
  • 应用场景：用来约束函数参数的类型

自带工具类型

  • 操作接口类型
    • Partial，将类型所有属性变为可选的
    • Required， 将给定类型的所有属性变为必选
    • Readonly，将给定类型属性设为只读
    • Pick，从给定类型中选出指定的键值
    • Omit，去除指定的键值后返回新类型
  • 联合类型
    • Exclude，从联合类型中去除指定类型
    • Extract，从联合类型中提取出指定类型
    • NonNullable，从联合类型中去除null或undefined类型
    • Record，生成接口类型
  • 函数类型
    • ConstructorParameters，获取构造函数的构造参数，需要使用infer关键字
    • Parameters，获取函数参数并返回
    • ReturnType，获取函数返回类型
    • ThisParameterType，获取函数this参数类型
    • ThisType，在对象字面量中指定this的类型
    • OmitThisParameter，去除函数类型中的this类型
  • 字符串类型
    • 模版字符串Uppercase、Lowercase、Capitalize，Uncapitalize

Express

Express 路由

Routing是 由一个URI 和 一个特定的HTTP方法 组成的，涉及到应用如何相应客户端对某个网站节点的访问。

结构组成: app.METHOD(PATH, HANDLER)

  • app是一个express实例
  • METHOD是某个HTTP请求方式中的一个
  • PATH是服务器端的路径
  • HANDLER是当路由匹配到时需要执行的函数

Express 托管静态文件

将静态资源文件所在的目录座位参数传递给express.static中间件就可以提供静态资源文件的访问了。

例如：
在public目录放置了静态资源，设置app.use(express.static(‘public’))
就可以通过localhost:3000/images/xxx.png访问了
还可以指定一个挂载前缀，app.use(‘/static’, express.static(‘public’))
访问路径为localhost:3000/static/images/xxx.png

文件路径是相对路径，即存放静态文件的目录名不会出现在url中

在express中404并不是一个error (404相对来说只是功能未完全实现)
因此错误处理器中间件捕获不到404，需要单独拎出来处理

设置错误处理器,多一个err参数

字符串模式的路由路径
?、+、*、()是正则表达式的子集

app.route() 创建路由路径的链式路由句柄

可用express.Router类创建模块化、可挂载的路由句柄

参考 http://www.expressjs.com.cn/starter/installing.html
