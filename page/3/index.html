ts基础类型

TypeScript 是 JavaScript 的类型的超集，它可以编译成纯 JavaScript。
即TS包含了JS的所有特性

常见类型

元祖类型 Tuple

表示一个已知元素数量和类型数组，各元素类型不必相同

当访问越界元素时，会使用联合类型替代

🚫 ts 3.1版本 之后，访问越界元素会报错

枚举类型

默认情况下，元素值从0开始，也可手动赋值

提供便利：由枚举值得到属性名

any类型 任意类型

对现有代码进行改写时，any类型十分有用

  • 允许你在编译时可选择地包含或移除类型检查
  • 并且当只知道一部分数据的类型时，any 类型也是有用的

void 无任何类型

通常用于函数，表示没有返回值
声明一个 void 变量是无意义的，只能赋给 null | undefined

null & undefined 任意类型

默认情况下 null 和 undefined 是所有类型的子类型
子类型可以赋值给父类型
像下面这个例子，null 和 undefined 赋值给 number类型 的变量

常规编译不会出错
严格编译 tsc xxx.ts –strictNullChecks 将会检查 null，从而规避一些问题

联合类型 任意类型

never 那些永不存在的值的类型

  • 是任意类型的子类型，可以赋值给任何类型
  • 别的类型都不是 never的子类型，不可赋值给 never
  • 常用于抛出一些异常 或 没有返回值的函数表达式 或 箭头函数表达式的返回值类型

返回never的函数 必须存在 无法达到的终点

object

除 number / string / boolean / symbol / null / undefined 之外的类型，非原始类型
类似object.create() API

类型断言

有时候你会遇到这样的情况，你会比 TypeScript 更了解某个值的详细信息。
通常这会发生在你清楚地知道一个实体具有比它现有类型更确切的类型。

通过类型断言这种方式可以告诉编译器，“相信我，我知道自己在干什么”。
类型断言好比其它语言里的 类型转换，但是不进行特殊的数据检查和解构。
它没有运行时的影响，只是在 编译阶段起作用

Express

Express 路由

Routing是 由一个URI 和 一个特定的HTTP方法 组成的，涉及到应用如何相应客户端对某个网站节点的访问。

结构组成: app.METHOD(PATH, HANDLER)

  • app是一个express实例
  • METHOD是某个HTTP请求方式中的一个
  • PATH是服务器端的路径
  • HANDLER是当路由匹配到时需要执行的函数

Express 托管静态文件

将静态资源文件所在的目录座位参数传递给express.static中间件就可以提供静态资源文件的访问了。

例如：
在public目录放置了静态资源，设置app.use(express.static(‘public’))
就可以通过localhost:3000/images/xxx.png访问了
还可以指定一个挂载前缀，app.use(‘/static’, express.static(‘public’))
访问路径为localhost:3000/static/images/xxx.png

文件路径是相对路径，即存放静态文件的目录名不会出现在url中

在express中404并不是一个error (404相对来说只是功能未完全实现)
因此错误处理器中间件捕获不到404，需要单独拎出来处理

设置错误处理器,多一个err参数

字符串模式的路由路径
?、+、*、()是正则表达式的子集

app.route() 创建路由路径的链式路由句柄

可用express.Router类创建模块化、可挂载的路由句柄

参考 http://www.expressjs.com.cn/starter/installing.html

搭建步骤

记录一下:)

  1. 前提：本地 node git 环境
  2. 安装hexo – 打开终端，输入 sudo npm install -g hexo
  3. 初始化hexo – 本地目录下创建名为hexo的文件夹，终端定位到该目录下，执行命令：nexo init
  4. 安装依赖 – npm install
  5. 生成静态页面 – hexo generate
  6. 开启服务 – hexo server 本地监听 localhost:4000
  7. 远程创建repository – your_user_name.github.io (与github用户名一致)
  8. 安装git部署器 – npm install hexo-deployer-git –save
  9. 修改配置文件 – 打开_config.yml 在文件末尾修改
     deploy
 type: git 
 repo: 你的repository地址 
       如 https://github.com/xxx/xxx.github.io.git
 branch: master
  10. 部署 – hexo deploy
  11. 效果 – 打开网站 https://xxx.github.io
  12. 发表文章 – hexo new ‘文章名’
  13. 编辑完，重新生成并部署 – hexo clean hexo g hexo d
  14. hexo在hexo目录下执行
