chrome devTools

chrome DevTools的运用

console

控制台复制变量到剪切板

右键 -> store object as global variable
得到temp变量 -> copy(temp) -> 剪切板上就有了temp


XHR重放

在Network Tab下选中对应的XHR记录，右键可以重放网络请求。
还可以复制请求包到命令行下修改后执行

截图

  • Capture area screenshot
    自定义选中截图区域
  • Capture full size screenshot
    截取HTML完整渲染图
  • Capture node screenshot
    截取某个DOM节点的渲染图，使用前得先在Elements Tab下选中DOM节点
  • Capture screenshot
    截取浏览器视窗所见屏幕

Performance

性能监控

  • 在调试某个web网页，想要查看从HTML渲染到屏幕的过程或运行时用户交互时的渲染、重绘、重拍，内存占用、页面变化过程。
  • 可以在Performace Tab下，点击红色小圆点即可录制页面的完整变化过程

Source Tab / DEBUG工具栏

Event Listener Breakpoints

事件监听断点，打开这个列表，在触发该事件时进入断点，调试器会停留在触发事件代码行。
展开事件列表，选择要监听的事件，勾选即可

Snippets

代码片段，开发者自己写一些调试代码的代码片段，比如解析json数据等工具函数。
在代码片段中的代码可以访问当前页面中的变量和函数，不会因刷新而丢失。

  • 右键即可run
  • 不需要的时候可以右键remove

Coverage

代码使用率（有效率）统计，支持运行时录制统计。

Rendering

监控页面重绘、重排时变化的区域进行高亮处理，还可查看帧率相关信息，用于网页性能优化

打开方式： 在more tools菜单中可选择Rendering工具

Layers

层，当一个页面足够复杂的时候，在css加持下，可能会出现一些渲染性能问题
这时候可以通过Layers工具显式地查看DOM层关系。
打开方式： 在more tools菜单中可选择Layers工具

移动端H5调试DEBUG

Chrome浏览器地址栏访问：chrome://inspect，可进入调试网页

参考资料：https://mp.weixin.qq.com/s/gBv8iF2JAp6VnjW7WsQ4MQ

性能优化总结

希望这次学习能构建一个完整的性能优化体系，而不是人云亦云

不急着去做，而是多思考
到底我们需要优化什么，哪些是关注的点

用什么衡量？ – 性能指标
设定指标后，优化到什么程度才合适？这就要确定 性能标准
然后开始性能实践，接着采集上报，这就出现监控预警平台

那么，

什么样的指标值得关注

  • 可衡量的，即可通过代码来度量
  • 关注以用户为中心的关键结果和真实体验

几个关键指标，几个极限值待补充

  • 加载：LCP (最大绘制内容)
  • 交互：CLS (视觉稳定性指标，即布局偏移量)
  • FID (首次输入延迟)

首屏时间=白屏时间+渲染时间

从页面加载过程来看3个阶段中有哪些性能瓶颈点

  • 客户端发起请求阶段
    当用户在浏览器输入URL时，浏览器向DNS服务器发起DNS查询，获取到IP地址，然后浏览器通过IP地址找到目标服务器，
    发起TCP三次握手和TLS协商，建立TCP连接

在这个过程中，本地缓存、DNS 查询、HTTP 请求容易成为瓶颈点
本地缓存可以让静态资源加载更快，一般包括强缓存和协商缓存两种形式
每进行一次DNS 查询，都需要经历手机到信号塔再到认证DNS服务器的过程，耗时过长，可以让DNS查询走缓存
HTTP 请求最大的瓶颈点在于请求阻塞，一般浏览器同域名的连接限制为6个，为了解决阻塞问题，可以通过不同域名增加请求并行
连接数，例如将静态服务器xxx.com分散为几个域名，每次请求随机选一个域名地址进行请求

  • 服务端数据处理请求阶段
    服务端接收到请求后，会有检验拦截，处理加工数据最后返回给前端
    那么这个过程中，需要关注是否做了数据缓存处理、是否做了 Gip 压缩，以及是否有重定向

  • 客户端页面渲染阶段
    前端拿到数据后，开始解析并渲染

所谓解析，就是 HTML 解析器把页面内容转换为 DOM 树和 CSSOM树的过程。
构建DOM树的过程中有3点会严重影响前端性能：

  • 当 HTML 标签不满足 Web 语义化时，浏览器就需要更多时间去解析 DOM 标签的含义
  • DOM 节点的数量越多，构建 DOM 树的时间就会变长，进而延长解析时间
